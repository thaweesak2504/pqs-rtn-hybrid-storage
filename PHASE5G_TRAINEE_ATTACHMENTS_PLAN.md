# แผนการดำเนินการ: ระบบแนบไฟล์อเนกประสงค์สำหรับคำตอบของ Trainee (Universal Trainee Attachments)

## 🎯 วัตถุประสงค์
ยกระดับช่องพิมพ์คำตอบของ Trainee (UserAnswers) ให้สามารถแนบไฟล์หลักฐานประกอบการตอบคำถามได้ทุกข้อ (ไม่จำกัดเฉพาะข้อ 3xx.1.1 และ 3xx.1.2) โดยไฟล์แนบทั้งหมดจะถูกเก็บรวมอยู่ในฐานข้อมูล `content.db` และโฟลเดอร์ Document เดียวกัน เพื่อให้การ Export/Import สมบูรณ์ 100%

---

## 🛠️ Phase 1: Database & Backend Architecture

### 1. ปรับปรุงตาราง `UserAnswers` (Database Migration)
เพิ่มคอลัมน์ใหม่ในตาราง `UserAnswers` เพื่อเก็บ Path ของไฟล์แนบในรูปแบบ JSON Array
```sql
ALTER TABLE UserAnswers ADD COLUMN attachments TEXT;
```
*รูปแบบข้อมูลที่จะเก็บ:* `["data/DOC123/trainee-attachments/IMG_UUID.png", "data/DOC123/trainee-attachments/DOC_UUID.pdf"]`

### 2. สร้างโครงสร้างจัดเก็บไฟล์ (Directory Structure)
สร้างโฟลเดอร์สำหรับเก็บไฟล์แนบของ Trainee โดยเฉพาะ แยกออกจากภาพคำถาม เพื่อให้จัดการง่าย
* Path: `data/<DocID>/trainee-attachments/`

### 3. พัฒนา Tauri API (Backend Rust)
สร้างคำสั่ง (Commands) สำหรับจัดการไฟล์แนบ:
- `upload_trainee_attachment`: อัปโหลดไฟล์ คัดกรองนามสกุลที่อนุญาต และคัดลอกไฟล์ไปยังโฟลเดอร์ `trainee-attachments` พร้อมตั้งชื่อใหม่กันซ้ำ (เช่น `<QuestionID>_<UserID>_<UUID>.<ext>`)
- `delete_trainee_attachment`: ลบไฟล์ออกจากระบบเมื่อ Trainee กดยกเลิก
- ปรับปรุงคำสั่ง `save_trainee_answer` และ `get_trainee_answers` ให้รองรับคอลัมน์ `attachments`

## 🖼️ Phase 2: ประเภทไฟล์ที่รองรับ และเงื่อนไข (File Types & Limits)

ระบบจะเปิดให้รองรับไฟล์ 4 ประเภทหลัก ดังนี้:

1. **รูปภาพ (Images):**
   - **นามสกุลที่รองรับ:** `.jpg`, `.jpeg`, `.png`, `.webp`
   - **การแสดงผล:** โชว์เป็นภาพย่อ (Thumbnail) กดคลิกเพื่อขยายดูรูปเต็ม (Image Viewer) เหมือนกับรูปภาพคำถาม

2. **เอกสาร (Documents):**
   - **นามสกุลที่รองรับ:** `.pdf` (ตามความประสงค์)
   - **การแสดงผล:** โชว์เป็นไอคอน PDF พร้อมชื่อไฟล์ เมื่อคลิกจะใช้คำสั่งเปิดไฟล์ PDF ด้วยโปรแกรมภายนอก (Default PDF Reader ของระบบปฏิบัติการ)

3. **วิดีโอ (Videos):**
   - **นามสกุลที่รองรับ:** `.mp4`, `.webm`
   - **เงื่อนไข:** ขนาดไฟล์สูงสุด **ไม่เกิน 50 MB** ต่อคลิป เพื่อป้องกันปัญหา Database บวมและกระทบการ Export
   - **การแสดงผล:** โชว์เป็น Video Player ขนาดย่อที่สามารถกดเล่น (Play) ได้ทันทีในหน้าจอ หรือคลิกเพื่อขยายดูเต็มจอ

4. **ไฟล์เสียง (Audio / Voice Recordings):**
   - **นามสกุลที่รองรับ:** `.mp3`, `.wav`, `.m4a`, `.ogg`
   - **วัตถุประสงค์:** สำหรับ Qualifier สั่งให้ Trainee อัดเสียงอธิบายความเข้าใจด้วยคำพูดตนเอง (เพื่อป้องกันการ Copy/Paste ข้อความมาส่ง)
   - **การแสดงผล:** โชว์เป็น Audio Player (`<audio controls>`) ที่สามารถกดฟังเสียงได้โดยตรงจากหน้าจอ

*หมายเหตุ: โควต้าจำกัดรวม แนบได้สูงสุด 3 ไฟล์ต่อข้อ (คละประเภทได้)*

## 🖥️ Phase 3: การปรับปรุงหน้าจอแสดงผล (UI Integration)

### 1. มุมมอง Trainee (Trainee View)
- ปรับปรุง Component `TraineeAnswerBox` เดิม
- เพิ่มปุ่ม **"📎 แนบไฟล์ (รูปภาพ/PDF/วิดีโอ/เสียง)"** ไว้ใต้กล่องข้อความ
- มีแถบแสดงสถานะการอัปโหลด และแสดงรายการไฟล์ที่แนบมาพร้อมปุ่มลบ (🗑️)
- สามารถเปิดฟังเสียง (`<audio>`) หรือดูวิดีโอ (`<video>`) ได้ทันทีเพื่อตรวจสอบความถูกต้องก่อนส่ง
- **กรณีพิเศษสำหรับ 3xx.1.1 และ 3xx.1.2:** จะซ่อนกล่องพิมพ์ Text ให้เหลือเพียงปุ่มแนบไฟล์อย่างเดียว

### 2. มุมมอง Qualifier (Qualifier View)
- ผู้ตรวจ (Qualifier) จะเห็นทั้งข้อความคำตอบและไฟล์แนบที่ Trainee ส่งมา
- สามารถกดดูรูปภาพ, ดูวิดีโอ, ฟังเสียงอธิบาย, หรือเปิด PDF ได้ทันที เพื่อประกอบการตัดสินใจ
- กดยืนยันสถานะ **"ผ่าน / ปรับปรุง"** ได้ตามกระบวนการปกติ

### 3. มุมมองผู้สร้างข้อสอบ (Question Editor)
- **การจัดการ Foreign Key อัตโนมัติ:** สำหรับข้อ 3xx.1.1 และ 3xx.1.2 ผู้สร้างจะไม่เห็นกล่อง "คำเฉลย" (ซ่อนทิ้ง) แต่ระบบหลังบ้านจะแอบสร้าง Dummy Answer Key ให้เพื่อรักษาความถูกต้องของ Database Constraints
