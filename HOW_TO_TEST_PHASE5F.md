# คู่มือการทดสอบระบบ (Manual Test) หลังจบ Phase 5F

การทดสอบนี้ออกแบบมาเพื่อตรวจสอบความถูกต้องของ Component ที่ถูกแยกออกมาจาก `QuestionFormCard.tsx` ได้แก่ 
1. `AnswerKeyEditor` (การจัดการเฉลย)
2. `ReferenceEditor` (การอ้างอิงเอกสาร)
3. `SubQuestionEditor` (การจัดการรายการคำถามย่อย)
4. `QuestionMetadataEditor` (การตั้งค่าคะแนน, จำนวนครั้ง, การละเว้น)

ขอให้คุณดำเนินการตาม Checklist ด้านล่างนี้ในแอปพลิเคชันของคุณ:

---

## 🟢 ส่วนที่ 1: การจัดการเฉลยและอ้างอิง (Section 100 / 200)

**เป้าหมาย:** ทดสอบการทำงานของ `AnswerKeyEditor` และ `ReferenceEditor`
- [ ] 1. ไปที่หมวด **Section 100** หรือ **Section 200**
- [ ] 2. กดสร้างคำถามใหม่ (หรือกดปุ่มแก้ไข "✏️" ในคำถามที่มีอยู่)
- [ ] 3. สังเกตที่ปุ่มไอคอนด้านขวาบนของแบบฟอร์ม กดปุ่ม **"รูปหนังสือ"** (เพิ่มการอ้างอิง)
- [ ] 4. ทดสอบเลือกเอกสารอ้างอิง ระบุหน้า (Page) หรือพิมพ์เอกสารอ้างอิงอื่นๆ (Custom)
- [ ] 5. ทดสอบการกดปุ่ม **"กุญแจ"** (เพิ่มเฉลย)
- [ ] 6. ลองพิมพ์ข้อความเฉลย และแนบรูปภาพในช่องเฉลย
- [ ] 7. กด **"บันทึก"** และสังเกตดูว่า ข้อมูลอ้างอิงและเฉลยถูกบันทึก/แสดงผลครบถ้วนหรือไม่

---

## 🟠 ส่วนที่ 2: การจัดการคำถามย่อย (Section 200)

**เป้าหมาย:** ทดสอบการทำงานของ `SubQuestionEditor` และระบบการละเว้น (ไม่ต้องอธิบาย)
- [ ] 1. ไปที่หมวด **Section 200**
- [ ] 2. กดปุ่มแก้ไข "✏️" ที่หัวข้อ **2xx.2**
- [ ] 3. ตรวจสอบว่ากล่อง **"รายการคำถามย่อย (SubQuestion List)"** แสดงขึ้นมาอย่างถูกต้อง
- [ ] 4. ลองเลือก **สาขาอาชีพหลัก** และ **สาขาย่อย**
- [ ] 5. ตรวจสอบรหัส Auto (ต้องขึ้นต้นด้วย `22...`)
- [ ] 6. ลองกด Checkbox "เลือกข้อย่อยที่ใช้งาน" หรือลองลบ/แก้ไขข้อมูลย่อย
- [ ] 7. ลองติ๊ก Checkbox **"ไม่ต้องอธิบาย"** (อยู่ขวาบนของฟอร์ม) สังเกตว่าช่องอธิบายและกล่อง SubQuestion ต้องซ่อนไป
- [ ] 8. กด **"บันทึก"** และไปแก้ไขข้อ **2xx.4** ให้ผลลัพธ์แบบเดียวกัน (รหัส Auto ต้องขึ้นต้นด้วย `24...`)

---

## 🟣 ส่วนที่ 3: ระบบคะแนนและการละเว้น (Section 300)

**เป้าหมาย:** ทดสอบการทำงานของ `QuestionMetadataEditor` (ส่วน Score และ Exempted)
- [ ] 1. ไปที่หมวด **Section 300**
- [ ] 2. ไปที่คำถามย่อยระดับ L2 (ตัวอย่าง: `3xx.2.1` เป็นต้น) แล้วกดแก้ไข "✏️"
- [ ] 3. จะต้องเห็นกล่องข้อความ **"จำนวนครั้งที่ต้องปฏิบัติ"** และกล่อง **"คะแนน"**
- [ ] 4. ลองกดเพิ่ม-ลดปุ่มจำนวนครั้ง (`+` / `-`) และกรอก **คะแนน/ครั้ง** 
- [ ] 5. ระบบต้องคำนวณคะแนนรวมอัตโนมัติให้เห็นชัดเจน (ตัวอย่าง: รวม 2 × 5 = 10 คะแนน)
- [ ] 6. ลองติ๊ก Checkbox **"ไม่ต้องปฏิบัติ"** สังเกตว่า ช่องกรอกคะแนนและกล่องรูปภาพ ต้องถูกซ่อนไปทั้งหมด
- [ ] 7. กด **"บันทึก"** และตรวจสอบความถูกต้องของคะแนน

---

## 🟡 ส่วนที่ 4: การเชื่อมโยง Section (Section Picker)

**เป้าหมาย:** ทดสอบกล่องแสดงตัวเลือกแบบผูกมัด Section
- [ ] 1. ในหมวด **Section 300** ให้มองหาข้อ `3xx.1.3`, `3xx.1.4` หรือ `3xx.1.5`
- [ ] 2. กดแก้ไข "✏️" ที่ข้อใดข้อหนึ่ง
- [ ] 3. จะต้องเห็นกล่องตัวเลือก **"เลือก Section ที่ต้องผ่าน"**
- [ ] 4. ทดลองเลือก/ยกเลิก Section จากรายการด้านล่าง
- [ ] 5. ลองระบุ **คะแนน** ของแต่ละ Section ระบบต้องสรุปยอด "รวม ... คะแนน" ที่ด้านล่าง
- [ ] 6. กด **"บันทึก"** และลองกดเข้ามาแก้ไขอีกครั้ง เพื่อดูว่าระบบจำ Section ที่เลือกไว้ได้หรือไม่

---
**💡 คำแนะนำ:** หากระหว่างทดสอบพบว่ามีปุ่มไหนกดไม่ได้ ค่าคะแนนคำนวณผิด หรือมี UI ซ้อนทับกัน สามารถแจ้งพิกัดข้อให้ผมทราบได้เลยครับ ผมจะรีบตรวจสอบ Props ที่เราเพิ่งแยกไปให้ครับ
