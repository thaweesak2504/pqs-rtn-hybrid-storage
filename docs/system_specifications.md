# System Specifications & Business Rules

## 1. Document ID Generation (หลักการกำหนดรหัสเอกสาร)

ระบบใช้ **RtnUnit Code** (รหัสหน่วยงาน) เป็นแกนหลักในการสร้างรหัสประจำเอกสาร (Document ID) จำนวน **12 หลัก** เพื่อระบุที่มาและประเภทของเอกสาร

### Structure (โครงสร้างรหัส)

รูปแบบ: `UUUUU` + `TT` + `L` + `SSS` + `F` (รวม 12 หลัก)

| ส่วนประกอบ | ชื่อ (Name) | ความยาว | คำอธิบาย (Description) | ตัวอย่าง (Example) |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **Unit Code** | 5 Digits | รหัสหน่วยงาน 5 หลักแรก (ตัด 00 ท้ายออก) | `22724` (จาก 2272400) |
| **2** | **Type** | 2 Digits | ประเภทเอกสาร <br> 10 = ทั่วไป (General) <br> 20 = เฉพาะ (Specific) | `10` |
| **3** | **Level** | 1 Digit | ระดับผู้ใช้งาน <br> 0 = สัญญาบัตร (Commissioned) <br> 1 = ประทวน (Non-commissioned) | `1` |
| **4** | **Sequence** | 3 Digits | ลำดับเอกสารที่สร้างของหน่วยนั้นๆ (Run Number 001-999) | `009` |
| **5** | **Format** | 1 Digit | รูปแบบการใช้งาน <br> 0 = คำถาม (Print Question) <br> 1 = คำตอบ (Print Answer) <br> 2 = ถาม-ตอบ (Desktop App) | `2` |

### Example (ตัวอย่างการใช้งาน)

**กรณีศึกษา**: กองโรงงานไฟฟ้าอาวุธ (2272400) สร้างเอกสาร PQS สำหรับนายทหารประทวน เป็นเล่มที่ 9 ของหน่วย ในรูปแบบ Desktop App ประเภททั่วไป

1.  **Unit Code**: ตัด 2 หลักท้ายของ `2272400` -> ได้ **`22724`**
2.  **Type**: ทั่วไป -> ได้ **`10`**
3.  **Level**: ประทวน -> ได้ **`1`**
4.  **Sequence**: เล่มที่ 9 -> ได้ **`009`**
5.  **Format**: Desktop App -> ได้ **`2`**

**Result (ผลลัพธ์)**:
*   **ID**: `227241010092`
*   **Thai Nums**: `๒๒๗๒๔๑๐๑๐๐๙๒`

## 2. Cover Page Generation (การสร้างหน้าปกอัตโนมัติ)

ระบบสามารถสร้างหน้าปก (Cover Page) โดยอัตโนมัติจาก **Document ID** และฐานข้อมูล **RtnUnits** โดยมีรูปแบบมาตรฐานดังนี้

### Layout Logic (หลักการจัดวาง)

| ส่วนประกอบ | แหล่งข้อมูล (Source) | รายละเอียด (Details) |
| :--- | :--- | :--- |
| **Logo** | Static / ID | ตราสัญลักษณ์กองทัพเรือ (อ้างอิงจากเลข 2 ตัวแรก) |
| **Header** | ID | ข้อความ "มกพ." ตามด้วยเลขไทยของ Document ID |
| **Title** | Static | ข้อความ "มาตรฐานกำลังพลกองทัพเรือ" |
| **Subtitle** | Static | ข้อความ "สำหรับ" |
| **Document Name** | Database | ชื่อเอกสาร (จาก Input ของ Creator หรือ Database) |
| **Unit (L3/L4)** | ID -> DB | ชื่อหน่วยระดับกอง/แผนก (Level 3 หรือ 4) ที่เป็นเจ้าของรหัส |
| **Unit (L2)** | ID -> DB | ชื่อหน่วยระดับกรม/สำนัก (Level 2) ที่หน่วยเจ้าของสังกัดอยู่ |
| **Format** | ID | ข้อความระบุรูปแบบ (เช่น "ถาม-ตอบ (Desktop App)") |

### Decoding Process (กระบวนการถอดรหัส)

ตัวอย่าง ID: `227222010212`

1.  **Extract Unit Code**: `22722`
2.  **Lookup Hierarchy**: ค้นหาหน่วยที่ขึ้นต้นด้วย `22722` ในฐานข้อมูล
    *   Found: `2272200` (กองทดสอบสรรพาวุธ) [Level 4] -> ใช้เป็น **Unit (L3/L4)**
    *   Find Parent: `2272000` (ศูนย์ซ่อมสร้างสรรพาวุธ) [Level 3] -> *Optional: แสดงควบคู่กันถ้ามี*
    *   Find Grandparent: `2270000` (กรมสรรพาวุธทหารเรือ) [Level 2] -> ใช้เป็น **Unit (L2)**
3.  **Extract Format**: Digits สุดท้าย `2` -> "ถาม-ตอบ (Desktop App)" -> ใช้เป็น **Format**
